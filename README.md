# Prototyping STEM Interactives with AI

## 1. Prompting with the COSTAR framework

1. Write a prompt for your STEM interactive using the COSTAR framework and paste it into a new Gemini chat.
2. Select Canvas from the Tools dropdown at the bottom left corner of the chat field, then hit Enter to generate your STEM interactive.
3. Continue entering prompts into Gemini to refine your STEM interactive.

| Element | Description | Sample |
| :--- | :--- | :--- |
| **Context (C)** | Essential background information that helps the AI understand the situation, environment, and circumstances surrounding your request. | I want students to investigate the relationship between light intensity and the rate of photosynthesis. In this experiment, a hydrilla plant is submerged in water. As it photosynthesizes, it produces oxygen bubbles. The independent variable is light intensity (manipulated by changing the distance of a lamp). The dependent variable is the rate of photosynthesis (measured by counting the number of bubbles produced per minute). |
| **Objective (O)** | The specific goal or desired outcome you want to achieve with your prompt. | Code me an interactive virtual lab where students can move a lamp towards/away from a hydrilla, observe animated bubble production, time the experiment with a stopwatch, log data into a table, and visualize the results on a scatter plot with a line of best fit. |
| **Style (S)** | How the AI should communicate and present information. E.g., Persona(s) the AI should adopt, language the AI should use. | Adopt the persona of a MOE STEM curriculum specialist, senior educational technologist and full-stack developer. |
| **Tone (T)** | The emotional quality and attitude that should be conveyed in the AI's response. | Encouraging but concise and educational, using age-appropriate scientific terminology. |
| **Audience (A)** | Who will receive, use or be affected by the AI's response. | Primary 5-6 students (ages 11-12) in Singapore. |
| **Response (R)** | The desired format, length, structure, and/or technical requirements of the deliverable that you want the AI to produce. Can also contain inclusion/exclusion criteria. | \*Format: A single standalone HTML file with JavaScript (logic) and Tailwind CSS (style).<br><br>\*Technical Requirements: Must run offline/locally, require no external dependencies, and be easily embeddable as a iframe/web widget.<br><br>**Inclusion Criteria**<br>\*Visual Simulation: A water-filled beaker with a green hydrilla plant. A movable lamp with a light beam that changes intensity/size based on distance. Bubbles should animate rising from the plant when the "lamp" is on.<br>\*Mathematical Model: As distance of the lamp increases, the rate of bubble production decreases based on the Inverse Square Law.<br>\*Controls: A slider to adjust lamp distance (10cm to 100cm) and a prominent "Start/Stop" button.<br>\*Measurement Tools: A digital stopwatch to track elapsed time in MM:SS format and a real-time automated bubble counter.<br>\*Data Logging: An automated results table that records distance, time, bubble count, and calculated rate (bubbles/min) each time a run is stopped.<br>\*Data Visualization: An interactive scatter plot (via Chart.js) that plots the "Average Rate of Bubble Production" against "Distance" and updates after each experiment run. Include a calculated Line of Best Fit (linear regression) once there are at least 3 data points.<br>\*Assessment:<br>- A "Conclusion" section with dropdown selections to test understanding of the relationship between light and photosynthesis.<br>- An open-ended "Bonus Question" with automated validation and feedback (specifically recognizing "Oxygen" or "O2").<br>\*State Management: "Reset" functionality to clear all recorded table data, graph points, and student answers.<br><br>**Exclusion Criteria**<br>\*No external database or server-side storage (session-based only).<br>\*No multi-page navigation; the entire lab must remain a single-page interface.<br>\*Focus strictly on light intensity. Do not vary any other biological variables (e.g., CO2 concentration or temperature). |
| **Refinement (R)** | Be specific!<br><br>Describe...<br>\*Component: What to change<br>\*Location: Where it is<br>\*Initial vs desired state: How you want it to be changed | Critically review the code and make the following refinements.<br><br>In the Simulation Window:<br>1. Make the water-filled beaker narrowed so that it is just wide enough to fit the hydrilla.<br>2. Rotate the lamp 90 degrees clockwise.<br>3. Place the water-filled beaker with the hydrilla on the left and the movable lamp towards the right. They should be middle-aligned, with the beam of the lamp shining towards the left where the hydrilla is.<br>4. The markings on the ruler should be at 10cm intervals instead of 25cm intervals.<br>5. The 0cm mark on the ruler should start beneath the hydrilla plant instead of at the leftmost edge of the Simulation Window. Keep the position of the 100cm mark unchanged. |

## 2. Configuring for SLS & adding scoring

1. Once you're satisfied, enter the prompts below into Gemini **one at a time** to make your STEM interactive SLS-compatible and add scoring.<br>
   *\*SLS compatibility will allow your STEM interactive to render correctly in an SLS activity. Scoring is for integration with the Interactive Response Assistant in later steps.*
2. In the Canvas pane to the right of the main chat, switch from Preview to Code view using the toggle button in the top right corner.
3. Click on the text field, select the entire code (Ctrl + A), copy (Ctrl + C) and paste (Ctrl + V) it into a new Notepad file.
4. Save this Notepad file, ensuring that "File name" is **index.html** and "Save as type" is changed to **All files**.

| Purpose | Sample Prompt |
| :--- | :--- |
| **SLS Compatibility** | **# PERSONA**<br>You are a MOE STEM curriculum specialist, senior educational technologist and full-stack developer.<br><br>**# CONTEXT**<br>My STEM interactive (code in the uploaded index.html) is not rendering correctly on MOE's Student Learning Space (SLS). My goal is to tweak the code (without changing the intent and features) to make my STEM interactive render correctly on SLS.<br><br>**# TASK**<br>1. Deeply study the uploaded index.html for the code of my STEM interactive.<br>2. Deeply study the technical requirements for html files to render correctly on SLS.<br>3. Based on Steps 1 and 2, without changing the intent and features of my STEM interactive, critically review and refine the code to make my STEM simulation render correctly on SLS.<br>4. Present me the code from Step 3 within the Gemini Canvas.<br><br>**# TECHNICAL REQUIREMENTS FOR SLS COMPATIBILITY**<br>IMPORTANT: Non-Negotiable Technical Boundaries<br>\*Self-Contained HTML5: Entire app < 500 MB.<br>\*No External Connections: No CDNs (e.g., Bootstrap, Tailwind via link), no external APIs, no Google Fonts. All CSS/JS must be internal.<br>\*Proportional Scaling: Strictly NO absolute pixels (px). Use %, vh/vw, em, rem. The app must work in an SLS \<iframe>.<br>\*Vector-First UI: Use inline SVGs for all graphics to ensure crisp scaling.<br>\*Offline-First: 100% functional without internet.<br>\*Client-Side Only: No server/database. Use localStorage for state. |
| **Scoring** | Critically review the code for my STEM interactive and make the following refinements without changing anything else.<br><br>For the Scientific Conclusion:<br>1. Add an automated validation feature for the dropdown selections, where each dropdown is marked either correct (score 1) or wrong (score 0). Update the score based on the latest dropdown selections by the student. But do not display the raw score to the student.<br>2. When the Bonus Question is checked, a correct answer should receive a score of 1 while a wrong answer should receive a score of 0. Update the score based on the latest answer given by the student on pressing the "Check" button. But do not display the raw score to the student. |

## 3. Adding xAPI integration for score & response collection in SLS

1. Upload "index.html", ["xAPI.js"](https://github.com/scbedodiv07-svg/vpl-advanced/blob/6d2a7fe6d46fb4af15a187d00114a292b4764f25/xAPI.js) and ["xapiwrapper.min.js"](https://github.com/scbedodiv07-svg/vpl-advanced/blob/6d2a7fe6d46fb4af15a187d00114a292b4764f25/xapiwrapper.min.js) to a new Gemini chat.
2. Select Canvas from the Tools dropdown at the bottom left corner of the chat field.
3. Paste the following prompt into the chat field.<br>
   *\*You may edit Step 3 according to the data you wish to collect from students' interactions (e.g. scores, questions, selections, responses, etc.)*
4. In the Canvas pane to the right of the main chat, switch from Preview to Code view using the toggle button in the top right corner.
5. Click on the text field, select the entire code (Ctrl + A), copy (Ctrl + C) and paste (Ctrl + V) it into your "index.html" file.
6. Save "index.html".

| To Collect | Sample Prompt |
| :--- | :--- |
| **Score and responses** | Instructions for Gemini:<br>**# PERSONA**<br>You are an expert coder, API integration specialist and experienced educator."<br>**# CONTEXT**<br>I have uploaded an "index.html" file containing the code of my STEM interactive for Student Learning Space (SLS). I want to integrate xAPI tracking into this STEM interactive using the helper libraries "xAPI.js" and "xapiwrapper.min.js". The goal is to collect the student's score and selections/responses when they interact with the STEM simulation, and then send it to SLS so that these data are visible as feedback in the Interactive Response Assistant feature of SLS.<br>**# TASK**<br>1. Deeply study the uploaded files to understand what the code does.<br>2. Add code into "index.html" to integrate it with the helper libraries for xAPI tracking.<br>3. Still in "index.html", identify where the score and user responses change, and modify the event handlers so that these data automatically save to SLS whenever the student changes their dropdown selections in the Scientific Conclusion section or presses the "Check" button for the Bonus Question. Each score and selection/response should then be visible as feedback in the Interactive Response Assistant feature of SLS.<br>4. Present me the updated code for "index.html" of my STEM interactive within the Gemini Canvas. |
| **Experimental data, score and responses** | Instructions for Gemini:<br>**# PERSONA**<br>You are an expert coder, API integration specialist and experienced educator.<br>**# CONTEXT**<br>I have uploaded an "index.html" file containing the code of my STEM interactive for Student Learning Space (SLS). I want to integrate xAPI tracking into this STEM interactive using the helper libraries "xAPI.js" and "xapiwrapper.min.js". The goal is to collect the student's experimental data, score and selections/responses when they interact with the STEM simulation, and then send it to SLS so that these data are visible as feedback in the Interactive Response Assistant feature of SLS.<br>**# TASK**<br>1. Still in "index.html", identify where experimental runs are recorded and the table results are logged, and modify the event handlers so that these data automatically save to SLS each time an experimental run is stopped. Each experimental run, together with the results for distance, time, bubble count and calculated rate (bubbles/min), should then be visible as feedback in the Interactive Response Assistant feature of SLS.<br>3. Also in "index.html", identify where the score and user responses change, and modify the event handlers so that these data automatically save to SLS whenever the student changes their dropdown selections in the Scientific Conclusion section or presses the "Check" button for the Bonus Question. ## 4. Preparing the files for upload to SLS
Select all 3 files and right click to compress them into a single ZIP file.<br>
*\*"index.html" contains the code for your STEM interactive. "xAPI.js" and "xapiwrapper.min.js" contain the code for the xAPI integration.*

## 5. Using your STEM interactive in an Interactive Response Question (IRQ)
1. In your SLS activity, select **Question** > **Free-Response** > **Interactive Response Assistant** to create a new IRQ and upload the ZIP file from above.
